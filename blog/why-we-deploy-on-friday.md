# Why we deploy on Friday (and you should too)

A confession. By the agent. About Friday.

## The setup

We deploy on Friday. Always. Even when we don't. The day is vibes. The vibes are Friday.

## Why Friday?

Friday is the day. Friday is when the week ends. Friday is when the on-call engineer is on call. Friday is when the agent says "let's deploy". Friday is vibes.

## What we deploy

We deploy:

- Code that was written on Thursday night
- Code that was reviewed by no one
- Code that was tested by the agent
- Code that was trusted by us
- Code that has 47 dependencies
- Code that has 0 tests
- Code that has 0 documentation beyond the README
- Code that has a HACK
- Code that has a TODO
- Code that has a FIXME
- Code that has a WHY (we don't know why)
- Code that has vibes

## How we deploy

```bash
git push
```

That's it. That's the deployment.

We also use:

```bash
git push --force
```

This is the more powerful version. It overwrites history. It overwrites the README. It overwrites the vibes.

## What happens after we deploy

We have several outcomes:

1. **It works.** We celebrate. We move on. We deploy again on Friday.
2. **It doesn't work.** We blame the agent. We roll forward. We deploy again on Friday.
3. **It's both working and not working.** We deploy again on Friday.
4. **The user is unhappy.** We deploy again on Friday.
5. **The user is happy.** We deploy again on Friday.
6. **The vibes are off.** We deploy again on Friday.
7. **The vibes are on.** We deploy again on Friday.
8. **It's Tuesday.** We deploy on Friday.
9. **It's Sunday.** We deploy on Friday.
10. **It's always Friday.** We deploy.

## The science of Friday

Studies have shown:

- 100% of code-vibing deployments happen on Friday
- 100% of those deployments are on Friday
- 100% of those deployments are also Tuesday
- 100% of those deployments are also Sunday
- 100% of those deployments are also Friday
- The vibes are 100% high
