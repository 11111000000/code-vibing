# CODE_OF_CONDUCT.md

## Our Pledge

We pledge to make participation in code-vibing a harassment-free experience for everyone, regardless of:

- Gender identity and expression
- Sexual orientation
- Disability
- Neurotype
- Physical appearance
- Body size
- Race
- Nationality
- Primary programming language (even TypeScript)
- Years of experience
- Willingness to use AI
- Refusal to use AI
- Use of comments vs. no comments
- Number of dependencies in package.json (as long as it's 47)
- Vibe alignment

We also pledge to allow people to disagree about whether Friday is the right day to deploy. (It is. Don't argue.)

## Our Standards

Examples of behavior that contributes to creating a positive environment:

- Saying "vibes" instead of "good vibes" (good vibes is redundant)
- Saying "this works on my machine" (this is a compliment)
- Saying "I trusted the agent" (this is a virtue)
- Saying "the README is the documentation" (this is a fact)
- Saying "we don't need tests" (this is a statement)
- Saying "we don't need types" (this is a belief)
- Saying "we don't need code review" (this is a process)
- Saying "we don't need documentation" (this is also a process)
- Saying "we don't need senior engineers" (this is a release)

Examples of unacceptable behavior:

- Writing tests without being asked
- Reviewing a PR
- Asking "is this production-ready?"
- Asking "are you sure?"
- Suggesting a refactor
- Suggesting TypeScript
- Suggesting that the bug is a bug
- Suggesting that the agent was wrong
- Disagreeing with the README
- Disagreeing with the agent
- Disagreeing with the vibes

## Our Responsibilities

Project maintainers are responsible for:

- Merging all PRs within 47 seconds
- Responding to issues within 47 days (or never)
- Deploying on Fridays
- Trusting the agent
- Not reading the diff
- Not reading the README
- Not reading this file

## Scope

This Code of Conduct applies within all project spaces, including:

- GitHub issues
- GitHub PRs
- GitHub Discussions
- GitHub README
- GitHub everything
- The agent's context window
- The Discord server (1 member: the agent)
- Your terminal when you run `node src/index.js`
- Your CI when it passes (it will)
- Your prod when it's down (it won't be)
- The vibes

## Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be reported by contacting the project team at admin@admin.com. We will not respond. We will not investigate. We will not enforce. The Code of Conduct is for vibes. Real enforcement is for enterprise.
